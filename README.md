## 📖 项目简介 (Introduction)
本项目是 **2024 吉比特未来游戏制作人大赛** 的参赛作品。
作为核心 Gameplay 程序员，我主要负责角色的 **3C 系统（Character, Camera, Control）** 开发与玩法代码架构搭建。旨在通过精细的物理参数调优与输入处理算法，在移动端实现“粘手”且精准的操作反馈。

---

## ✨ 技术亮点 (Technical Highlights)
### 1. 3C 手感优化 
为了解决传统物理跳跃的“漂浮感”和“操作延迟”，实现了一套定制化的运动控制器：
* **Coyote Time (土狼时间)：** 允许玩家在离开平台后的 **0.1s** 内依然可以执行跳跃指令，大幅提升容错率。
* **Input Buffer (输入缓冲)：** 实现了一个 **0.15s** 的指令队列，允许玩家在落地前预输入跳跃指令，落地瞬间自动执行，保证连贯性。
* **Jump Cut (跳跃截断)：** 根据按键按下的时长动态调整跳跃高度，实现“小跳”与“大跳”的精准控制。

### 2. 状态驱动的 MVC 架构 (State-Driven MVC)
为了应对复杂的角色状态（如：空闲、奔跑、跳跃、冲刺、受击、死亡），抛弃了臃肿的 if-else 堆叠，设计了一套清晰的分层架构：

* **Model (数据层)** *： 使用 ScriptableObject 存储所有配置数据（如移动速度、跳跃力、无敌时间），实现了策划/美术的免代码调参。

* **View (表现层)** *： 负责动画状态机 (Animator) 的同步与特效播放，与逻辑层完全解耦。

* **Controller (逻辑层)** *： 基于 FSM (有限状态机) 管理所有状态切换，确保逻辑的严谨性与可扩展性。

### 3. 高性能事件中心 (Event System)
构建了一套基于 泛型  与 C# 扩展方法 的轻量级消息总线，替代了传统的 SendMessage。

零装箱 ： 优化了值类型参数传递，避免了不必要的 GC 开销。

类型安全： 事件类型动态注册，编译期检查错误，降低了模块间的耦合度。
